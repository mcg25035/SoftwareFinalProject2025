# FOMO 線上購物網站系統 - 系統介紹

## 📋 系統概述

FOMO (Fear Of Missing Out) 線上購物網站系統是一個功能完整的電子商務平台，根據 IEEE 830 與 ISO/IEC 25010 軟體工程標準設計開發。系統採用模組化架構，包含四個核心子系統，提供完整的購物、支付、管理功能。

---

## 🏗️ 系統架構

### 核心子系統

1. **資料庫系統 (DBS 1.4.0)**
   - 核心資料模型管理
   - 資料持久化與關聯
   - 資料驗證與完整性

2. **顧客系統 (CS 1.2.0)**
   - 商品瀏覽與搜尋
   - 購物車管理
   - 訂單管理
   - 個人資料管理
   - 商品收藏與評價

3. **金流系統 (PS 1.4.0)**
   - 支付處理
   - 交易記錄
   - 退款管理

4. **管理者系統 (AS 1.1.0)**
   - 商品管理
   - 訂單管理
   - 退款審核
   - 使用者管理
   - 系統日誌

## 🚀 快速開始

### 1. 安裝依賴
```bash
pip install -r requirements.txt
```

### 2. 資料庫遷移
```bash
python manage.py migrate
```

### 3. 初始化資料
```bash
python manage.py init_fomo_data
```

### 4. 建立管理者
```bash
python manage.py createsuperuser
```

### 5. 啟動伺服器
```bash
python manage.py runserver
```

### 6. 訪問系統
- **商品列表**: http://localhost:8000/customer/products/
- **管理後台**: http://localhost:8000/administrator/dashboard/
- **Django Admin**: http://localhost:8000/admin/

## 📝 功能特色

### 1. 完整的購物流程
從商品瀏覽 → 加入購物車 → 結帳 → 付款 → 訂單追蹤，提供完整的購物體驗。

### 2. 多種付款方式
支援信用卡、銀行轉帳、貨到付款等多種付款方式。

### 3. 完善的退款機制
顧客可申請退款，管理者可審核處理。

### 4. 商品評價系統
購買過的顧客可對商品進行評價，幫助其他顧客做出購買決策。

### 5. 收藏功能
顧客可收藏喜歡的商品，方便日後查看。

### 6. 通知系統
系統會自動發送訂單、支付、退款等相關通知。

### 7. 管理後台
功能完整的管理後台，方便管理者管理商品、訂單、退款等。

### 8. 系統日誌
記錄所有重要操作，便於審計與問題追蹤。

## ⚠️ 注意事項

### 開發環境
- 目前使用 SQLite 資料庫（適合開發測試）
- DEBUG 模式開啟（方便開發除錯）

### 生產環境建議
1. 更換 SECRET_KEY
2. 關閉 DEBUG 模式
3. 設定 ALLOWED_HOSTS
4. 使用 PostgreSQL 或 MySQL
5. 設定 HTTPS
6. 整合真實支付 API
7. 設定靜態檔案服務
8. 使用 Gunicorn + Nginx

### 功能限制
- 支付功能目前為模擬實作，需整合第三方支付 API（如綠界、藍新等）
- 未實作即時通知（WebSocket）
