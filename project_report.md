## 2. 專案特色

本系統為一個完整的 B2C 電子商務平台，設計目標為提供使用者流暢的購物體驗與管理者高效的營運工具。主要特色包含：

### 顧客導向功能
*   **完整購物流程**：提供從商品瀏覽、搜尋、篩選、加入購物車到結帳的流暢體驗。
*   **互動與個人化**：支援商品收藏、評價系統，以及個人化的通知中心，讓使用者隨時掌握訂單動態。
*   **會員中心**：提供完整的個人資料管理、歷史訂單查詢與地址管理功能。

### 營運管理功能
*   **視覺化儀表板**：管理者後台首頁提供即時營收、訂單數據與熱門商品排行，輔助決策。
*   **全方位管理**：具備商品上下架、庫存管理、訂單狀態追蹤與退款審核功能。
*   **安全性設計**：實作系統日誌 (System Logs) 記錄關鍵操作軌跡，並採用 RBAC (Role-Based Access Control) 權限控管。

### 金流與架構
*   **模擬金流系統**：完整模擬信用卡與轉帳付款流程，並包含退款處理邏輯。
*   **標準化架構**：資料庫設計符合 IEEE 830 標準，確保資料一致性與正規化。

## 3. 專案使用技術與架構

### 技術棧 (Tech Stack)
*   **後端框架**: Python Django 5.2.1
*   **前端框架**: Bootstrap 5.3.0 (RWD 響應式設計)
*   **資料庫**: SQLite (開發環境)
*   **版本控制**: Git
*   **架構模式**: MTV (Model-Template-View)

### 系統架構
本系統採用模組化設計，將功能拆分為四個獨立的 Django App，以降低耦合度並提升維護性：

1.  **DBS (資料庫系統)**：定義核心資料模型 (Category, Product, Order, UserProfile 等)。
2.  **CS (顧客系統)**：處理前台購物邏輯 (瀏覽、購物車、結帳、會員中心)。
3.  **PS (金流系統)**：負責支付交易處理與退款管理。
4.  **AS (管理者系統)**：提供後台管理介面 (儀表板、商品/訂單/使用者管理)。

## 附錄：系統安裝與操作手冊

### 環境需求
*   Python 3.12 或以上版本
*   Git

### 安裝步驟

1.  **複製專案 (Clone Repository)**
    ```bash
    git clone https://github.com/mcg25035/SoftwareFinalProject2025.git
    cd SoftwareFinalProject2025
    ```

2.  **建立虛擬環境 (建議)**
    ```bash
    python -m venv venv
    # Windows
    venv\Scripts\activate
    # macOS/Linux
    source venv/bin/activate
    ```

3.  **安裝依賴套件**
    ```bash
    pip install -r requirements.txt
    ```

4.  **資料庫遷移**
    ```bash
    python manage.py migrate
    ```

5.  **建立管理者帳號 (Superuser)**
    ```bash
    python manage.py createsuperuser
    ```

6.  **啟動伺服器**
    ```bash
    python manage.py runserver
    ```
    伺服器啟動後，請在瀏覽器開啟 `http://127.0.0.1:8000/`。

### 操作說明
*   **前台入口 (顧客)**：`http://127.0.0.1:8000/`
    *   可註冊新帳號或瀏覽商品。
    *   測試購物流程：商品 -> 加入購物車 -> 結帳 -> 選擇模擬信用卡 -> 完成訂單。
*   **後台入口 (管理者)**：`http://127.0.0.1:8000/administrator/dashboard/`
    *   需使用 Superuser 帳號登入。
    *   可查看儀表板、管理商品、訂單與退款。
