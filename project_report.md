## 2. 專案特色

本系統為一個完整的 B2C 電子商務平台，設計目標為提供使用者流暢的購物體驗與管理者高效的營運工具。主要特色包含：

### 顧客導向功能
*   **完整購物流程**：提供從商品瀏覽、搜尋、篩選、加入購物車到結帳的流暢體驗。
*   **互動與個人化**：支援商品收藏、評價系統，以及個人化的通知中心，讓使用者隨時掌握訂單動態。
*   **會員中心**：提供完整的個人資料管理、歷史訂單查詢與地址管理功能。

### 營運管理功能
*   **視覺化儀表板**：管理者後台首頁提供即時營收、訂單數據與熱門商品排行，輔助決策。
*   **全方位管理**：具備商品上下架、庫存管理、訂單狀態追蹤與退款審核功能。
*   **安全性設計**：實作系統日誌 (System Logs) 記錄關鍵操作軌跡，並採用 RBAC (Role-Based Access Control) 權限控管。

### 金流與架構
*   **模擬金流系統**：完整模擬信用卡與轉帳付款流程，並包含退款處理邏輯。
*   **標準化架構**：資料庫設計符合 IEEE 830 標準，確保資料一致性與正規化。

## 3. 專案使用技術與架構

### 技術棧 (Tech Stack)
*   **後端框架**: Python Django 5.2.1
*   **前端框架**: Bootstrap 5.3.0 (RWD 響應式設計)
*   **資料庫**: SQLite (開發環境)
*   **版本控制**: Git
*   **架構模式**: MTV (Model-Template-View)

### 系統架構
本系統採用模組化設計，將功能拆分為四個獨立的 Django App，以降低耦合度並提升維護性：

1.  **DBS (資料庫系統)**：定義核心資料模型 (Category, Product, Order, UserProfile 等)。
2.  **CS (顧客系統)**：處理前台購物邏輯 (瀏覽、購物車、結帳、會員中心)。
3.  **PS (金流系統)**：負責支付交易處理與退款管理。
4.  **AS (管理者系統)**：提供後台管理介面 (儀表板、商品/訂單/使用者管理)。

## 5. 專案成果

本專案成功建置了一套功能完整的 B2C 電子商務平台「FOMO 購物網站」，系統整合了前後端互動、資料庫管理及金流模擬。透過模組化的開發策略，我們將系統劃分為資料庫系統 (DBS)、顧客系統 (CS)、金流系統 (PS) 與管理者系統 (AS) 四大核心部分，各部分成果詳述如下：

1. DBS (Data Base System) 資料庫系統
本系統採用 Django ORM 建立關聯式資料庫，成功定義並實作了支撐電商運作的完整資料模型，確保資料的一致性與關聯性：
- 核心模型建置：完成了商品 (Product)、分類 (Category)、訂單 (Order) 與使用者 (UserProfile) 等基礎模型。
- 擴充功能支援：設計了客服工單 (SupportTicket) 與商品問答/評價 (ProductQA/Review) 模型，支援複雜的前後端互動需求。
- 資料關聯性：建立了商品與分類的多對一關係，以及訂單與訂單細項 (OrderItem) 的一對多關係，確保交易資料的完整紀錄。

2. CS (Customer System) 顧客系統
顧客系統作為前台互動核心，提供了流暢的使用者體驗 (UI/UX)，涵蓋從瀏覽、會員管理到售後服務的完整流程：
- 商品瀏覽與檢索：實作了全站商品列表，支援「價格高低」與「商品評價」排序，並具備側邊欄分類篩選功能。商品頁面亦整合了自動化「相似商品推薦」機制。
- 會員中心：完成了會員註冊與登入驗證（含密碼強度提示），並提供個人資料管理頁面，允許使用者即時更新配送資訊。
- 互動與支援：除了基本的購物車操作外，會員可針對特定商品提問或查看評價。此外，系統內建「客服工單系統」，會員可提交問題並追蹤處理進度（如：已解決、處理中），實現雙向溝通。

3. PS (Payment System) 金流系統
金流系統模擬了真實電商的交易環節，處理訂單生成、支付狀態變更及退款邏輯：
- 結帳與訂單生成：成功串接配送資訊填寫與訂單確認流程，訂單成立後系統會自動生成唯一識別編號。
- 多元支付模擬：支援信用卡、轉帳與貨到付款選項。專案中特別展示了信用卡支付流程，付款完成後，訂單狀態會即時更新。
- 退款管理機制：使用者可於歷史訂單中發起「退款申請」，系統會記錄申請原因並將狀態同步至後台供管理員審核。

4. AS (Administrator System) 管理者系統
基於 Django Administration 開發的高權限後台，賦予管理員對網站數據的完全控制權，實現高效的網站維運：
- 商品 CRUD 管理：管理員可輕鬆新增、編輯或刪除商品，包含上傳圖片與設定庫存，所有變更皆會即時同步至前台。
- 訂單監控：後台提供訂單列表檢視功能，可查看所有交易金額與支付狀態，並處理用戶提交的退款申請。
- 客戶服務整合：管理員可直接在後台回覆會員的「客服工單」與「商品提問」，回覆內容將即時顯示於前台會員介面，大幅提升客服效率。

### 結語
本專案成功實作了一個架構完整、功能齊全的電商平台。透過 Django 強大的生態系，我們在有限時間內完成了從資料庫設計、前後端開發到權限管理的完整流程，並學習到正規化資料庫設計與 MTV 架構的實務應用。

3.  **即時通訊**：實作 WebSocket 支援即時客服對話與庫存預警通知。

## 附錄：系統安裝與操作手冊

### 環境需求
*   Python 3.12 或以上版本
*   Git

### 安裝步驟

1.  **複製專案 (Clone Repository)**
    ```bash
    git clone https://github.com/mcg25035/SoftwareFinalProject2025.git
    cd SoftwareFinalProject2025
    ```

2.  **建立虛擬環境 (建議)**
    ```bash
    python -m venv venv
    # Windows
    venv\Scripts\activate
    # macOS/Linux
    source venv/bin/activate
    ```

3.  **安裝依賴套件**
    ```bash
    pip install -r requirements.txt
    ```

4.  **資料庫遷移**
    ```bash
    python manage.py migrate
    ```

5.  **建立管理者帳號 (Superuser)**
    ```bash
    python manage.py createsuperuser
    ```

6.  **啟動伺服器**
    ```bash
    python manage.py runserver
    ```
    伺服器啟動後，請在瀏覽器開啟 `http://127.0.0.1:8000/`。

### 操作說明
*   **前台入口 (顧客)**：`http://127.0.0.1:8000/`
    *   可註冊新帳號或瀏覽商品。
    *   測試購物流程：商品 -> 加入購物車 -> 結帳 -> 選擇模擬信用卡 -> 完成訂單。
*   **後台入口 (管理者)**：`http://127.0.0.1:8000/administrator/dashboard/`
    *   需使用 Superuser 帳號登入。
    *   可查看儀表板、管理商品、訂單與退款。
